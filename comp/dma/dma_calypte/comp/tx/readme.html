TX DMA Calypte¶

ENTITY TX_DMA_CALYPTE IS¶

Warning

The Completer Completion interface is not supported yet. Calypte Controller supports only Memory Write PCIe transactions.

This is transmitting part of the DMA Calypte core. The major structure can be changed by setting the CHANNEL_ARBITER_EN generic parameter to either true or false. This parameter enables/disables the CHANNEL_ARBITER component which merges streams from all CHANNEL_CORE components. The output interface with enabled arbiter remains the same but valid data are transmitted on the interface no. 0 only. When CHANNEL_ARBITER is disabled, each CHANNEL_CORE has its own separate output. The block scheme of the TX DMA Calypte controller is provided in two following figures (N = <number of channels> - 1):

=====

Setting of internal parameters of CHANNEL_CORE

=====

=====

FIFO_DEPTH

natural

512

FIFO depth in number of items stored. The overall capacity in bytes can be obtained when this number is multiplied by the width of the PCIe CQ bus in bytes. The FIFO should fit at least one packet of the PKT_SIZE_MAX size. Otherwise a halt of the whole controller can occur.

CHANNELS

natural

8

Set the number of DMA channels, each channel is controlled by one CHANNEL_CORE component.

=====

Enabling of CHANNEL_ARBITER component

=====

This component merges output streams from all CHANNEL_CORE components. When enabled, the valid data occur only on the 0th USR_TX_MFB interface. Then disabled, each channel has its own output.
